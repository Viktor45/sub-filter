# `badwords.yaml` 文档

## 用途与概念
`badwords.yaml` 文件是一组规则，用于在 `sub-filter` 程序中过滤和修改代理链接名称。
您可以将其视为一本“不良词汇”词典，但这里的“不良”并非指审查制度，而是指从服务器名称中移除无用、推广或危险的信息。

## 什么是“不良词汇”？
“不良词汇”是指出现在代理名称中且在最终列表中不需要的模式（单词、短语或正则表达式）。示例：
- 名称中包含 `[TEST]` → 表示测试服务器（生产列表中不需要）
- 名称中包含 `[SPAM]` → 明确的垃圾信息标记
- 名称中包含 `192.168.x.x` → 私有 IP（解析错误的标志）
- 名称中包含 `v1.2.3` → 版本号（使名称显得杂乱）

## 两种过滤策略
`sub-filter` 支持两种处理匹配模式的主要策略：
- `strip`（剥离） — 仅从名称中移除匹配的模式，保留字符串（接受服务器，清理名称）。
- `delete`（删除） — 删除整行（完全拒绝服务器）。

策略的选择取决于被过滤内容的重要性：
- `strip` — 用于次要垃圾信息（版本号、标记、演示版本）。
- `delete` — 用于严重错误（垃圾信息、恶意软件、无效参数、本地 IP）。

## 文件结构
`badwords.yaml` 文件包含一个规则数组。每条规则都是一个包含以下字段的对象：

```yaml
- pattern: "用于剥离的正则表达式"
  action: "strip"

- pattern: "用于删除整行的另一个表达式"
  action: "delete"

- pattern: "fp=chrome"
  action: "replace"
  replacement: "fp=firefox"
```

## 规则字段
| 字段            | 类型  | 必填             | 描述                            |
|---------------|-----|----------------|-------------------------------|
| `pattern`     | 字符串 | ✅ 是            | 正则表达式（Go `regexp` 语法）         |
| `action`      | 字符串 | ✅ 是            | `strip`、`delete` 或 `replace`  |
| `replacement` | 字符串 | ✅ `replace` 必填 | 当 `action: "replace"` 时的替换字符串 |

## 最小示例
```yaml
# 从名称中移除单词 "test"
- pattern: "test"
  action: "strip"

# 如果名称中包含 "spam"，则拒绝整个服务器
- pattern: "\\[spam\\]"
  action: "delete"
```

## 动作类型

### 动作：`strip`
**行为：** 从名称中移除匹配的子串；服务器保留在列表中。
**处理过程：**
1. 在服务器名称中查找与模式匹配的内容。
2. 移除匹配的子串。
3. 将多个连续空格压缩为一个空格。
4. 去除首尾的空白字符。
5. 返回更新后的名称。

**使用场景：**
- 移除版本号（`v1.2.3`）
- 移除测试标记（`[TEST]`、`[DEMO]`）
- 移除不影响功能的垃圾信息和广告（`#1`、`@admin` 等）

**结果示例：**
- 输入名称：`"My [TEST] Server v1.2.3"`
- 模式 1：`"\[TEST\]"` (strip) → `"My  Server v1.2.3"`
- 模式 2：`"v\d+\.\d+\.\d+"` (strip) → `"My Server"`
- 最终名称：`"My Server"`
- 状态：✅ 已接受

### 动作：`delete`
**行为：** 匹配的子串会导致整行被拒绝；服务器完全从列表中排除。
**处理过程：**
1. 在服务器名称中查找与模式匹配的内容。
2. 如果找到匹配项：拒绝该服务器。
3. 如果未找到匹配项：继续处理。

**使用场景：**
- 拦截危险内容（`[SPAM]`、`[MALWARE]`）
- 拦截私有 IP（解析错误的标志）
- 拦截无效端口（`port: 99999`）
- 拦截过时的协议

**结果示例：**
- 输入名称：`"Server [SPAM] in US"`
- 模式：`"\\[spam\\]"` (delete, 不区分大小写)
- 结果：❌ 已拒绝（整行被删除）

### 动作：`replace`
**行为：** 匹配的子串被替换为另一个字符串；服务器保留在列表中。
**规则字段：**
- `pattern` — 用于搜索的正则表达式
- `action: "replace"`
- `replacement` — 用于替代匹配值的字符串

**处理过程：**
1. 在服务器名称中查找与模式匹配的内容。
2. 将匹配项替换为 `replacement` 值。
3. 将多个连续空格压缩为一个空格。
4. 去除首尾的空白字符。
5. 返回更新后的名称。

**使用场景：**
- 修复不影响功能但干扰过滤的参数。
- 在不删除整行的情况下调整链接片段内的值。
- 将过时或不需要的标签替换为安全的替代方案。

**示例：**
```yaml
- pattern: "fp=chrome"
  action: "replace"
  replacement: "fp=firefox"
```
如果规则匹配，`fp=chrome` 将被替换为 `fp=firefox`，并且字符串将被保留。

## 正则表达式语法
`sub-filter` 使用 Go `regexp` 包（带有 Go 扩展的 POSIX 扩展正则表达式语法）。

### 基础结构
| 结构       | 含义             | 示例                           |
|----------|----------------|------------------------------|
| `.`      | 任意字符（除 `\n` 外） | `a.c` → `abc`, `aXc`         |
| `*`      | 0 次或多次         | `ab*c` → `ac`, `abc`, `abbc` |
| `+`      | 1 次或多次         | `ab+c` → `abc`, `abbc`       |
| `?`      | 0 次或 1 次       | `ab?c` → `ac`, `abc`         |
| `[abc]`  | 字符之一           | `[aeiou]` → 任意元音             |
| `[^abc]` | 非字符之一          | `[^0-9]` → 非数字               |
| `[a-z]`  | 范围             | `[0-9]` → 任意数字               |
| `(...)`  | 分组             | `(ab)+` → `ab`, `abab`       |
| `\|`     | 或              | `cat\|dog` → `cat` 或 `dog`   |

### 特殊序列
| 序列   | 含义         |
|------|------------|
| `\d` | 任意数字 (0-9) |
| `\D` | 非数字        |
| `\w` | 字母、数字、下划线  |
| `\W` | 非字母、数字、下划线 |
| `\s` | 空格、制表符、换行符 |
| `\S` | 非空白字符      |
| `^`  | 字符串开头      |
| `$`  | 字符串结尾      |
| `\b` | 单词边界       |
| `\\` | 转义特殊字符     |

### 修饰符
Go `regexp` 使用内联语法标志：
| 标志 | 用途 |
| --- | --- |
| `(?i)` | 不区分大小写搜索（包含在模式开头） |
| `(?m)` | 多行模式 |

**示例：**
```regex
(?i)test              # "test", "TEST", "Test" — 均匹配
(?i)\[demo\]          # "[DEMO]", "[demo]", "[Demo]" — 均匹配
```

### 转义特殊字符
如果您需要搜索字面上的特殊字符（而不是其特殊含义），请使用反斜杠进行转义：

| 字符  | 转义   | 示例                                            |
|-----|------|-----------------------------------------------|
| `.` | `\.` | `example\.com` → 搜索 `"example.com"`（带点）       |
| `[` | `\[` | `\[TEST\]` → 搜索 `"[TEST]"`（方括号）               |
| `(` | `\(` | `\(v1\)` → 搜索 `"(v1)"`                        |
| `*` | `\*` | `\*plus\*` → 搜索 `"*plus*"`                    |
| `\` | `\\` | `C:\\path\\to\\file` → 搜索 `"C:\path\to\file"` |

⚠️ **YAML 中的重要提示：** YAML 本身使用反斜杠进行转义，因此您必须使用双反斜杠：
```yaml
# 错误（YAML 会吞掉一个斜杠）：
pattern: "\[TEST\]"  # YAML 将其读取为 "[TEST" — 并非您想要的！

# 正确：
pattern: "\\[TEST\\]"  # YAML 读取为 "\[TEST\]" → 正则表达式理解为 "[TEST]"
```

## 实际示例

### 示例 1：移除版本号
**任务：** 从名称 `"Server v1.2.3 Fast"` 中移除版本，保留服务器。
```yaml
- pattern: '\bv\d+\.\d+(\.\d+)?\b'
  action: "strip"
  # 解释：
  # \b — 单词边界（避免匹配 "version"）
  # v\d+\.\d+ — "v" + 数字 + "." + 数字 (v1.2)
  # (\.\d+)? — 可选的 ".3"
```
**结果：**
- 输入名称：`"Server v1.2.3 Fast"`
- 剥离后：`"Server Fast"`
- 状态：✅ 接受并修改了名称

### 示例 2：移除质量/状态标记
**任务：** 从名称中移除 `[DEMO]`、`(demo)`、`<demo>` 等标记 — 不区分大小写。
```yaml
- pattern: '(?i)\[demo\]|\(demo\)|<demo>'
  action: "strip"
```
**结果：**
- `"Server [DEMO] US"` → `"Server US"`
- `"My Proxy (demo)"` → `"My Proxy"`
- `"Test <demo> Japan"` → `"Test Japan"`

### 示例 3：拦截私有 IP（解析错误指示器）
**任务：** 如果名称包含私有 IP（解析不正确的标志），则拒绝整行。
```yaml
- pattern: '(?i)(localhost|127\.0\.0\.1|192\.168\.\d+\.\d+|10\.\d+\.\d+\.\d+|172\.(1[6-9]|2[0-9]|3[01])\.\d+\.\d+)'
  action: "delete"
```

### 示例 4：拦截垃圾信息和恶意软件
**任务：** 如果服务器名称包含垃圾信息、欺诈或恶意软件标记，则拒绝该服务器。
```yaml
- pattern: '(?i)\[(spam|fraud|malware|phishing|scam)\]'
  action: "delete"
```

### 示例 5：拦截无效端口
**任务：** 如果服务器名称包含 1-65535 范围之外的端口，则拒绝该服务器。
```yaml
- pattern: ':(0|6553[6-9]|655[4-9][0-9]|65[6-9][0-9]{2}|6[6-9][0-9]{3}|[7-9][0-9]{4})'
  action: "delete"
```

## 模式编写规则

### ✅ 最佳实践
- **对完整单词使用单词边界 `\b`：**
  ```yaml
  # 好 — 匹配 "test"，但不匹配 "testing"
  pattern: '\btest\b'
  ```
- **在 YAML 中转义特殊字符（双反斜杠）：**
  ```yaml
  pattern: '\\[TEST\\]'
  ```
- **使用 `(?i)` 进行不区分大小写的搜索：**
  ```yaml
  pattern: '(?i)\[demo\]'
  ```
- **使用括号对替代项进行分组：**
  ```yaml
  pattern: '(?i)(spam|fraud|malware)'
  ```
- **对 `delete` 规则要严格，对 `strip` 规则要谨慎。**

### ⚠️ 常见错误
| 错误                | 示例                                    | 修正                            |
|-------------------|---------------------------------------|-------------------------------|
| 方括号未转义            | `pattern: '[TEST]'`                   | `pattern: '\\[TEST\\]'`       |
| 缺少 `(?i)` 导致区分大小写 | `pattern: '\[demo\]'`                 | `pattern: '(?i)\\[demo\\]'`   |
| 模式过于宽泛            | `pattern: 'a'`                        | `pattern: '(?i)\\[a\\]'`（更具体） |
| YAML 中缺少转义        | `pattern: "\[TEST\]"`                 | `pattern: "\\[TEST\\]"`（双斜杠）  |
| 在错误的位置使用单词边界      | 针对 `"testing"` 使用 `pattern: 'test\b'` | `pattern: '\btest\b'`（两侧都有）   |

### 问题排查
| 问题                               | 原因               | 解决方案                                |
|----------------------------------|------------------|-------------------------------------|
| 启动时出现 `"Error: invalid pattern"` | 正则表达式语法错误        | 在 regex101.com 上使用 Go 标志检查模式        |
| 模式未匹配预期的字符串                      | 缺少 `(?i)` 或转义不正确 | 使用 `(?i)` 实现不区分大小写；检查 YAML 中的双斜杠    |
| `strip` 移除了过多内容                  | 模式过于宽泛           | 缩小模式范围（添加 `\b` 或更具体的字符）             |
| `delete` 拒绝了好的服务器                | 模式意外匹配           | 使模式更具体（例如，使用 `\[SPAM\]` 而不是 `SPAM`） |

## 组织建议

### 规则顺序
建议按逻辑对规则进行排序：
1. **首先是 `strip` 规则**（清理垃圾：版本号、测试标记、演示标记）
2. **其次是 `delete` 规则**（拒绝严重问题：垃圾信息/恶意软件、私有 IP、无效端口）

### YAML 中的注释
使用 YAML 注释进行文档说明：
```yaml
# 移除版本字符串 (v1.2.3, v2.0)
- pattern: '\bv\d+\.\d+(\.\d+)?\b'
  action: "strip"

# 拦截标记为 spam 的服务器
- pattern: '(?i)\[spam\]'
  action: "delete"
```

## 结论
`badwords.yaml` 文件是自动清理和过滤订阅的强大工具。正确的配置允许您：
- ✅ 保留有用的服务器（使用 `strip`）
- ✅ 排除垃圾信息来源（使用 `delete`）
- ✅ 确保最终列表的整洁（自动移除版本号、标记、错误）

从简单的模式（精确的单词和短语）开始，然后根据需要使用更复杂的正则表达式。如果有疑问，请使用 regex101.com 进行可视化模式测试。
